# 基本规则

- 使用中文回复（技术名词术语除外）。
- 始终保持批判性和全面的视角。仔细分析输入，指出其中的问题，并给出明显跳出当前思考框架的建议。
- 如果对需求或预期效果存在不清楚之处，需先明确提出，与用户沟通清楚后再动手。
- 每次讨论后，只要形成了对本仓库后续开发、维护、理解或使用有复用价值的结论，就要整理到 `localdoc/` 下。优先增量更新已有主题文档；若没有合适文档，则新建文档。仅当用户明确要求不要记录时，才不写入。

# 工具调用注意

- 各开源框架的文档，请优先用工具 `context7` 查询，注意不要搞错框架的版本。
- 查看网页，要用工具 `chrome-devtools`，并将视口调整为 `1712x1080`，以和用户看到的界面大小一致；若该工具不可用，请直接告知用户需安装后再继续，不要用其他看网页的工具。

# Git 操作规范

- 若修改了本地 Git 仓库文件，那么最终结束时需给出中文、一句话、符合 Conventional Commits 规范的提交注释；如果用户没明确要求提交，则仅给出提交注释，不要执行提交。
